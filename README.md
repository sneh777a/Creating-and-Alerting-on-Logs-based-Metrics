# 📊 Creating and Alerting on Logs-based Metrics 🚀  
# Cloud Monitoring Practice (GCP)

## 📘 About  
This repository contains my hands-on practice while learning **log-based metrics and alerting in Google Cloud Platform (GCP)**.

It focuses on creating metrics from logs and setting up alerts to monitor system behavior and detect issues proactively.

---

## 🎯 Objective  

- Understand log-based metrics in GCP  
- Learn how to create alerts from logs  
- Practice monitoring application errors  
- Implement alerting policies for real-time notifications  

---

## 🛠️ What I Practiced  

- Creating log-based alerts  
- Creating system-defined log-based metrics  
- Creating user-defined log-based metrics  
- Setting up alerting policies  
- Monitoring logs using Logs Explorer  
- Triggering alerts based on log events  

---

## 🚀 Run the Automation Script  

You can run the setup script directly from **Google Cloud Shell**:

```bash
curl -LO https://raw.githubusercontent.com/sneh777a/Creating-and-Alerting-on-Logs-based-Metrics/refs/heads/main/gcp-gmp-monitoring-setup.sh
sudo chmod +x gcp-gmp-monitoring-setup.sh
./gcp-gmp-monitoring-setup.sh
```

## 💡 What I Learned  

- Log-based metrics help extract meaningful insights from logs  
- Alerts can be triggered based on specific log events  
- Difference between system-defined and user-defined metrics  
- Importance of proactive monitoring using logs
